# Contributing

Thanks for considering contributing! This project follows a simple workflow to
keep changes easy to review.

## Quick Start

1. Fork the repository and create a feature branch.
2. Make small, focused commits.
3. Add or update tests/docs when behavior changes.
4. Open a pull request describing the change and rationale.

## Development Notes

- PHP 8.4 is required.
- Symfony 7.4 is the minimum supported version.
- Keep public APIs backwards compatible unless a breaking change is agreed.

## Code Style

- Follow existing code style and structure.
- Use strict types and explicit return types where possible.

## Reporting Issues

Please include:
- steps to reproduce
- expected vs actual behavior
- PHP/Symfony versions
- any relevant logs or stack traces
